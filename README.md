```jsx
const MongoClient = require("mongoDB").MongoClient;

let db; // 저장할 디비
MongoClient.connect("연결주소", function (에러, client) {
  db = client.db("데이터베이스명");

  db.collection("컬렉션명").insertOne(
    { 제목: "어쩌구" },
    function (err, result) {
      console.log("저장완료");
    }
  );
});
```

## 🎯 배운 것

### 1. body-parser

- 보통 req.body 부분을 undefined로 처리하는데
  여기서 `**body-parser**` 를 이용하면 해당 데이터를 사용자가 원하는 형태로 파싱하여 사용가능!

  ````jsx
  const express = require('express');
  const bodyParser = require('body-parser');
  const app = express();

      // 추가
      app.use(bodyParser().json()); // json 형태로 파싱
      app.use(express.urlencoded(){ extended : false })
      app.post('/', function(req,res){
      	let post = req.body ;
      	console.log(post);
      })
      ```

      `**app.use(express.urlencoded(){ extended : false })**`

      → false : 기본으로 내장된 **querystring 모듈** 사용

      → true : 따로 설치가 필요한 qs 모듈을 사용하여 쿼리 스트링을 해석
  ````

### API

- 웹서버와 고객간의 소통방법
- 어떻게 해야 서버랑 통신을 할수 있을까

### rest API

1. **uniform interface → 제일 중요!!**
   - 하나의 자료는 하나의 url로
   - url 하나를 알면 둘을 알수 있어야한다
   - 요청과 응답은 정보가 충분히 들어있어야한다.
2. client - server 역할 구분
   - 브라우저는 요청만 할뿐
   - 서버는 응답만 할뿐
3. stateless
   - 요청1과 요청2는 의존성이 없어야한다.
4. cacheable
5. layered system
6. code on demand

### nodemon → node monitor 의 약자

- 코드를 변경할 때 마다 자동으로 서버를 재실행해주는 도구

### 1. location.href 와 location.replace의 차이점

**href**

- 새로운 페이지로 이동
- 속성
- 기록된다
- location.href = ‘abc.php’

**replace**

- 기존 페이지를 새로운 페이지로 이동
- 메서드
- 기록되지 않는다.
- location.replace(’abc.php’)

### 2. innerHTML / innerText / textContent 의 차이

- innerHTML → html 그대로 출력
- innerText → 공백이 여러개라도 한개로 축소함. 보이는 부분만
- textContent → 숨겨진 텍스트까지 가져온다

### 3. scrollTo(x,y) , scrollBy(x,y) 의 차이

**scrollBy(x,y)**

- 상대적 위치
- 현재 위치를 기준으로 파라미터 값만큼 이동
- 창을 상대적인 좌표로 스크롤

```jsx
window.scrollBy(100, 100);
```

**scrollTo(x,y)**

- 절대적 위치
- 왼쪽 상단 모서리부터 시작
- 창을 절대적인 좌표로 이동

```jsx
window.scrollBy(100, 100);
```

  마진 병합 현상은 인접하는 블록요소의 상하단의 마진이 병합되는 현상
  마진의 크기는 병합되는 마진 중에서 큰 값을 가진 마진의 값으로 병합된다.

  1. **형제 지간의 마진 병합**

     margin-bottom과 margin-top 중 **숫자가 큰 값**으로 적용

     ```html
     <div class="box1">Hello World</div>
     <div class="box2">Hello World</div>
     ```

     ```css
     .box1 {
       margin-bottom: 150px;
     } /* 적용값 */
     .box2 {
       margin-top: 100px;
     }
     ```

  2. **부모 자식간의 마진 병합**

     자식 뿐만 아니라 부모에도 영향을 미침

     ```html
     <main role="main">
       <article></article>
     </main>
     ```

     ```css
     article {
       width: 200px;
       height: 200px;
       margin-top: 100px;
     }
     ```

     자식인 `<article>`뿐만 아니라 부모인 `<main>` 에도 영향을 미침

  3. **파일의 권한 변경** → chmod

     ```bash
     $ chmod [파일권한][변경할 파일 위치 또는 이름]
     ```

  4. **소유권 변경** → chown

     ```bash
     $ chown [소유할유저]:[소유할그룹][소유권 변경하고싶은 디렉토리 또는 파일명]
     ```

- **파일 명령어**

  1. **디렉토리 생성 → mkdir**

     ```bash
     $ mkdir [생성할 디렉토리 이름]
     $ mkdir [디렉토리 경로/디렉토리명]
     ```

  2. **빈파일 생성 → touch**

     ```bash
     $ touch [생성할 파일 이름]
     $ touch [디렉토리 경로/파일 명]
     ```

  3. **디렉토리 삭제 → rmdir**

     - 비어있는 디렉토리를 삭제
     - 지정한 디렉토리에 파일이 있을 경우 삭제 불가

     ```bash
     $ rmdir [삭제할 디렉토리 이름]
     ```

  4. **파일 및 디렉토리 삭제 → rm**

     ```bash
     $ rm [option][삭제할 파일 및 디렉토리 명]
     ```

     - option
       -r : 디렉토리와 그 하부 파일까지 삭제
       -f : 삭제 여부를 묻지 않고 바로 삭제
       -i : 삭제할 것인지 확인
       -rf : 삭제여부를 묻지 않고 하부파일이 있는 디렉토리까지 삭제

  5. **파일 및 디렉토리 복사 → cp**

     ```bash
     $ cp [옵션][대상 위치 및 이름][복사할 위치]
     ```

     - option
       -r : 하위 디렉토리와 파일 전체 복사
       -p : 소유주 , 그룹, 권한, 시간 정보를 그대로 복사

  6. **파일 및 디렉토리 이동 → mv**

     ```bash
     $ mv [대상위치/이름][이동하고싶은 위치/이름]
     ```

  7. **파일 내용 출력 → cat**

     ```bash
     $ cat [option][파일 이름]
     ```

     - option

       -n : 왼쪽에 줄번호와 함께 내용을 출력한다.

       `>` : 파일의 내용을 덮어쓴다.

       `> >` : 파일의 내용이 있다면 뒤에 내용 추가

  8. **파일 검색 → find**

     ```bash
     $ find [파일경로][-name][파일이름][-type d/f]
     ```

     -type d : 디렉토리

     -type f : 파일

  9. **명령어 옵션 확인 → —help**

     ```bash
     $ [사용하려는 명령어] --help
     ```

### 한 일

### 배운 것

- 자바스크립트 콘솔에서 입력받는 방법 - **readline**

  - readline 모듈을 이용하면 콘솔을 통해 값을 입력받을 수 있다.

  ```jsx
  // 1. 모듈 가져오기
  const readline = require("readline");

  // 2. **readline** 모듈을 이용해 입출력을 위한 인터페이스 객체 생성
  const rl = readline.createInterface({
    input: process.stdin,
    output: process.stdout,
  });

  // 3. rl 변수
  rl.on("line", (line) => {
    // 한 줄씩 입력받은 후 실행할 코드
    // 입력된 값을 line에 저장된다.
    rl.close(); // 필수! close가 없으면 입력을 무한히 받는다.
  });

  rl.on("close", () => {
    // 입력이 끝난 후 실행할 코드
    process.exit();
  });
  ```

### 느낀 점

- 예전에 자바를 배웠을때 콘솔에 입력받는 법을 배웠었는데 오늘 앨리스 강의를 들으면서 자바스크립트에서 콘솔창에 입력받는 법을 알았다.

  ```jsx
  // reduce((a,c) ⇒ { (a+c) , [초기값 : 지정안하면 첫번째 요소값]})
  // a는 누적값 , c는 현재값
  [1,2,3,4].reduce((a,c)=> (a+c));
  // a : 1 , c : 2
  // a : 3 , c : 3
  // a : 6 , c : 4
  // (return값) 10

  // 객체식으로도 표현가능하다
  ['가','나','다','라'].reduce((a,c,i) => { a[i] = c ; return a }, {})
  {0: '가', 1: '나', 2: '다', 3: '라'}

  //{0: '가'} '가' , i:0
  //{0: '가', 1: '나'} '나', i:1
  //{0: '가', 1: '나', 2: '다'} '다' i:2
  //{0: '가', 1: '나', 2: '다', 3: '라'} '라' i:3
  ```

## 1. 알고리즘

### 한일

1. 알고리즘 기초 강의를 듣고 기본적인 알고리즘 시스템에 대해 강의 및 실습을 해보았다.

### 배운 것

- 프로그램이 가장 작은 단위는 일반적으로 입력 → 처리 → 출력 의 단계를 거친다.
- 여기서 처리 단계가 알고리즘으로 보면 된다.

1. 입력(input) : 자료 구조(data Structure)가 담당하는 영역
   - 간단히는 변수 및 배열의 데이터가 사용되고 더 나아가 컬렉션, 파일, db의 데이터가 사용되는 영역
2. **처리(process) : 알고리즘 영역**
3. 출력(output) : ui가 담당하는 영역, 일반적으로 콘솔, 데스크톱, 웹, 모바일 등의 영역으로 나뉘어 가공된 데이터가 출력

- 난이도
  | 난이도 | 알고리즘 | 사용 유형 |
  | ------------------------------ | ------------------------------- | ------------------------------------------- |
  | 초급 | 합계(sum) | 합계 출력 |
  | | 개수(count; 횟수, 건수) | 자료 건수 출력 |
  | | 평균(AVERAGE) | 평균을 출력 |
  | | 최댓값(MAX) | 최댓값을 출력 |
  | | 최솟값(MIN) | 최솟값을 출력 |
  | 중급 | 최댓(소)값 → 최솟(대)값 | ~에 대해 최댓(소)값을 구하되, 동일값 발생시 |
  | ~에 대해 최솟(대)값을 구하시오 |
  | | 최대(소)값 → 최대(소)값 | ~에 대해 최대(소)값을 구하되, 동일값 발생시 |
  | ~에 대해 최대(소)값을 구하시오 |
  | 고급 | 근삿값(NEAR) | ~에 가까운 값을 구하시오 |
  | | 순위(RANK) | 순위를 구하시오 |
  | | 정렬(SORT) : 오름차순, 내림차순 | 오름(내림)차순을 정렬하시오 |
  | | 검색(SEARCH) | 특정 자료를 검색하시오 |
  | | 병합(MERGE) | 2개의 배열을 하나의 배열로 합치시오 |
  | | 최빈값(mode) | 가장 빈도수가 높은 자료를 구하시오 |
  | | 그룹(group) | 특정 항목별 그룹화하여 소계를 구하시오 |
